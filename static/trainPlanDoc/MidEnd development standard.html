技术研发二十三条军规

第一条          包名统一使用小写，点分隔符之间有且仅有一个自然语义的英语单词，包名统一使用单数形式。

第二条          类名使用UpperCameCase风格，必须遵从驼峰形似，但以下情形例外：（领域模型的相关命名）DO/BO/DTO/VO/DAO

第三条          类名如果有复数含义，类名可以使用复数形式。

第四条          定义DO/DTO/VO等POJO类时，不要加任何属性默认值。

反例：POJO类的gmtCreate默认值为new Date()，但是这个属性在数据提取时并没有置入具体值，在更新其他字段时又附带更新了此字段，导致创建时间被修改成当前时间。

第五条          抽象类命名使用Abstract或Base开头。异常类命名使用 Exception 结尾；测试类命名以它要测试的类的名称开始，以 Test 结尾。

第六条          方法名，参数名，成员变量，局部变量都统一使用lowerCamelCase,必须遵从驼峰形式。

第七条          常量命名应该全部大写，单词间用下划线隔开，力求语义表达完整清楚，不要嫌名字长。

第八条          所有的类注释的内容包括类的简述、作者、引用的接口、时间。

示例如下：

/**

* 附件组件的管理

* @author 张三

* @see 引用指标类(indicatorService)

* @since 2018-11-14

*

*/

public interface JxFjbService extends BaseService {

}

第九条          类，类属性，类方法的注释必须使用javadoc规范，使用/**内容**/格式，不得使用//内容 方式和/*内容*/方式。及时清理不再使用的代码段或配置信息。

第十条          方法内部单行注释，在被注释语句上方另起一行，使用//注释，方法内部多行注释使用/**/注释，注意与代码对齐。

第十一条          所有的覆盖方法，必须加@Override注解

说明：加@Override可以准确判断是否覆盖成功，另外如果在抽象类中对方法签名进行修改，对应的实现类会马上报编译错误。

第十二条          所有的枚举类型字段必须要有注释，说明每个数据项的用途。

第十三条          关于基本数据类型与包装数据类型的使用标准如下：

所有的抽象方法（包括接口中的方法）必须要用javadoc注释，除了返回值、参数、异常说明外，还必须指出该方法做什么事情，实现什么功能，尽量使用包装数据类型。

第十四条          在if/else/for/while/do语句中必须使用大括号，即使只有一行代码，避免使用下列的形式：if(condition) statement;

第十五条          Object的equals方法容易抛空指针异常应使用常量或确定有值的对象来调用equals。

第十六条          SimpleDateFormat是线程不安全的类，一般不要定义为static变量，如果定义 为static变量，必须加锁，或者使用DateUtils工具类或Jode-Time下的Datetime类。

第十七条          集合初始化时，指定集合初始大小。非分页情况下如果能够预估大小时尽量指定集合准确的大小。

第十八条          事务场景中，抛出异常被catch后，如果需要回滚，一定要手动回滚事务。

第十九条          不允许任何魔法值（即未经定义的常量）直接出现在代码中。

第二十条          单个方法的总行数不超过80行。

第二十一条          除常用方法（如getXxx/isXxx）等外，不要在条件判断中执行复杂的语句，将复杂逻辑判断的结果赋值给一个有意义的布尔变量，以提高可读性,尽量不要使用is开头的布尔变量名.

第二十二条          POJO类必须写toString方法，使用工具类source->generate toString时，如果继承了另一个POJO类，注意在前面加一下super.toString。

说明：在方法执行抛出异常时，可以直接调用POJO的toString()方法打印其属性值，便于排查问题。

第二十三条          循环体内，字符串的联接方式，使用StringBuilder的append方法进行扩展。

说明：反编译出的字节码文件显示每次循环都会new出一个StringBuilder对象，然后进行append操作，最后通过toString方法返回String对象，造成内存资源浪费。

反例：

String str = "start";

for (int i = 0; i < 100; i++) {

str = str + "hello";

}
